# Lab Summary

The main challenge was getting set up with Cohere instead of Anthropic: creating a Cohere API key, figuring out the right model ID to use (`command-r-08-2024`), and learning the Python SDK's syntax for calling it, since Cohere's `ClientV2.chat()` shapes its request and response differently than OpenAI's or Anthropic's (messages go in the same way, but the reply comes back as a list of typed content blocks, and usage is reported under `billed_units` rather than flat `input_tokens`/`output_tokens`). I solved this by digging into Cohere's docs to confirm the exact syntax, then isolating that provider-specific parsing inside a single `ask_cohere()` function so the rest of the pipeline only ever deals with a normalized `{response, usage}` shape regardless of provider. I learned that most of the friction in a multi-provider setup is at the integration boundary; once one provider's response is normalized to match the other's shape, the fallback and cost-tracking logic doesn't care which provider actually served the request. For improvement, I'd add a small integration test that hits each provider's real API once (behind a flag) just to catch SDK/response-shape changes early, rather than relying entirely on mocks that assume the shape is still correct.
